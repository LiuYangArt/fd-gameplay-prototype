# 自动测试与验证工具方案

更新时间：2026-02-17

## 当前已接入

1. TypeScript 严格类型检查
- 命令：`pnpm typecheck`
- 价值：提前发现跨模块接口不一致和空值问题。

2. 核心玩法单元测试（Vitest）
- 命令：`pnpm test`
- 价值：快速验证事件驱动链路、回合推进、伤害结算与胜负判定。

3. 构建可用性检查（Vite Build）
- 命令：`pnpm build`
- 价值：保证可测试版本可正常打包。

4. 一键回归
- 命令：`pnpm verify`
- 串联：类型检查 + 单测 + 构建。

## 下一阶段建议（可玩后再加）

1. Playwright 冒烟测试
- 目标：验证页面加载、关键按钮可触发、战斗能走完一回合。

2. 玩法回放快照测试
- 目标：记录命令序列并断言事件序列，保证重构不改变核心行为。

3. 数据校验脚本
- 目标：在导入技能/数值表时做 schema 校验，避免坏数据进入测试版本。
