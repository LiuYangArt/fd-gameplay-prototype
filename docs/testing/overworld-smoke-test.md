# 探索阶段最小冒烟验证（2026-02-25）

## 前置条件

1. 执行 `pnpm dev` 启动页面。
2. 浏览器打开后，确认右侧 HUD 显示 `Runtime Phase: Overworld`。

## 步骤 1：键鼠移动与跑步

1. 按住 `W/A/S/D` 移动角色，移动鼠标调整方向。
2. 按住 `Shift` 后继续移动。

预期结果：

- 胶囊体角色能稳定朝镜头方向移动。
- 按住 `Shift` 时速度明显快于默认行走。

## 步骤 2：手柄输入一致性

1. 连接手柄。
2. 左摇杆移动角色，右摇杆调整方向。
3. 按住 `RT` 持续移动。

预期结果：

- 左/右摇杆与键鼠行为一致。
- `RT` 可触发跑步速度。

## 步骤 3：遇敌切换战斗并回退

1. 主动接触任意蓝色倒金字塔敌人。
2. 进入战斗后按 `Enter` 或手柄 `A` 完成战斗操作。
3. 战斗结束后观察是否自动返回探索。

预期结果：

- 接触敌人后可在 2 秒内进入战斗态。
- 战斗结束后自动回探索，玩家回到安全点 `(0, 0)`。
- 刚触发战斗的敌人会从探索场景中移除。

## 步骤 4：F3 Debug 配置持久化

1. 按 `F3` 打开 Debug 菜单。
2. 调整 `Target Arm Length (cm)`，观察镜头远近变化。
3. 将 `Camera Lag Speed` 调到 `0`，持续移动角色观察镜头跟随。
4. 将 `Camera Lag Speed` 调低到 `2` 左右，持续移动角色观察拖尾效果。
5. 将 `Camera Lag Max Distance (cm)` 从 `300` 调到 `80`，观察最大拖尾距离被限制。
6. 点击“导出 JSON”，复制文本；刷新页面后确认配置保留。
7. 粘贴 JSON 后点击“导入 JSON”。

预期结果：

- `Target Arm Length` 调整后相机远近立即变化。
- `Camera Lag Speed=0` 时镜头几乎即时跟随。
- 降低 `Camera Lag Speed` 后可观察到明显拖尾。
- 降低 `Camera Lag Max Distance` 后拖尾距离明显受限。
- 刷新后仍保持上次配置。
- 导入 JSON 后可恢复同一配置。

## 步骤 5：命令验证

1. `pnpm lint`
2. `pnpm test`
3. `pnpm verify`

预期结果：

- 三条命令均通过。
