## 2. 现象与影响

- 用户/测试者看到的现象：
  - 3D 视口“看起来是 16:9”，但出现左右未撑满、上下留白、画面内容被拉伸。
  - 第三人称相机与角色朝向关系不稳定，角色可出现“朝向摄像机”的违和状态。
  - 鼠标与手柄右摇杆的上下控制方向与预期不一致。
  - 调试面板参数单位不统一，难以对齐 UE 迁移。
- 首次发现时间：2026-02-25
- 复现条件：
  - 桌面浏览器下调整窗口尺寸或布局区域尺寸后，观察 3D 画布显示比例与内容几何比例。
  - 键鼠/手柄混合测试相机俯仰与角色朝向。
  - 在 F3 Debug 面板调节相机距离、偏移、移动速度参数。
- 业务影响（阻塞测试、误导结论、数据污染等）：
  - 阻塞探索手感评估与输入方案确认。
  - 导致“是否满足 UE 迁移约束”的判断失真。
  - 增加反复改动次数，降低迭代效率。

## 4. 根因分析

- 直接根因：
  - 只在 CSS 层约束容器比例，未在 canvas 实际像素尺寸变化时同步 Babylon Engine 渲染尺寸，导致“容器比例正确但内容被拉伸/留白”。
  - 第三人称控制映射中，输入轴与相机/角色旋转的坐标约定未完全对齐（含鼠标与手柄俯仰方向）。
  - Debug 参数未统一标注 UE 语义单位，调参值可读性与迁移一致性不足。
- 深层根因（流程、设计、测试、工具）：
  - 缺少“容器尺寸变化 -> 渲染缓冲同步 -> 视觉比例复核”的固定检查步骤。
  - 缺少键鼠/手柄一致性的最小冒烟清单（尤其是右摇杆 Y 轴与鼠标 Y 轴同向约定）。
  - 缺少跨层单位规范（逻辑层、渲染层、HUD 标注）的一致性约束。
- 为什么之前的防线没有拦住：
  - 现有自动化验证以 lint/typecheck/test/build 为主，未覆盖视口比例与输入手感的视觉回归。
  - 文档层面此前未明确“UE 迁移时统一单位”的硬规则。

## 5. 修复方案

- 临时止血：
  - 调整样式，保证视口容器以宽度优先计算 16:9，避免重复高度约束造成留白。
- 永久修复：
  - 在 `USceneBridge` 中引入 `ResizeObserver` 监听 canvas 盒模型尺寸变化，触发 `engine.resize()` 同步渲染缓冲。
  - 修正第三人称相机输入映射，统一鼠标与手柄右摇杆俯仰方向语义。
  - 统一 Debug 面板参数单位并补充显示标注（`cm`、`cm/s`、`deg`），并在 `AGENTS.md` 明确 UE 单位规范。
- 关联代码变更：
  - `packages/web-client/src/game/USceneBridge.ts`
  - `packages/web-client/src/styles.css`
  - `packages/web-client/src/input/UInputController.ts`
  - `AGENTS.md`
- 风险与回滚方案：
  - 风险：`ResizeObserver` 触发频繁可能带来额外 resize 开销。
  - 缓解：仅在尺寸变化时调用 `engine.resize()`，并在 `Dispose()` 断开 observer。
  - 回滚：若出现兼容性问题，可回退到 `window.resize` 监听实现并保留当前样式修复。
