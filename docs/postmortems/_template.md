# Postmortem 模板

创建文件名：
`docs/postmortems/YYYY-MM-DD-<主题>.md`

## 1. 基本信息

- 日期：
- 负责人：
- 影响范围：
- 严重级别：`S1` / `S2` / `S3`
- 状态：`处理中` / `已修复` / `已验证`

## 2. 现象与影响

- 用户/测试者看到的现象：
- 首次发现时间：
- 复现条件：
- 业务影响（阻塞测试、误导结论、数据污染等）：

## 3. 时间线

1. `HH:MM` 发现问题
2. `HH:MM` 完成定位
3. `HH:MM` 完成修复
4. `HH:MM` 完成回归验证

## 4. 根因分析

- 直接根因：
- 深层根因（流程、设计、测试、工具）：
- 为什么之前的防线没有拦住：

## 5. 修复方案

- 临时止血：
- 永久修复：
- 关联代码变更：
- 风险与回滚方案：

## 6. 防再发动作（必须可执行）

1. 新增或补强的回归测试：

- 文件路径：
- 覆盖场景：

2. 新增或补强的规则/流程：

- lint/test/checklist/hook 的变更：
- 执行责任人：
- 截止时间：

## 7. 验证记录

- 本地验证命令：
- 验证结果：
- 复测人：

## 8. 经验沉淀

- 是否需要更新 `AGENTS.md`：
