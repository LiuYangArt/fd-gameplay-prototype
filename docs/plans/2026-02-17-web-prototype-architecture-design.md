# TS 网页玩法原型架构设计（面向 UE5 迁移）

## 目标

- 使用 TypeScript 快速验证回合制 3D 战斗玩法。
- 让测试者通过浏览器直接体验。
- 验证通过后，将核心玩法逻辑迁移到 UE5 + puerts，尽量避免重写。

## 架构分层

1. `gameplay-core`（纯逻辑层）

- 不依赖 DOM、渲染引擎、浏览器 API。
- 负责回合状态机、命令处理、事件派发、数值结算。
- 作为未来迁移到 UE 的主要复用层。

2. `web-client`（表现与输入层）

- 负责 3D 表现、UI 展示、键鼠/手柄输入采集。
- 将输入动作转换为核心层命令。
- 将核心层事件转换为画面与 UI 更新。

3. `ue-bridge`（迁移适配层）

- 预留与 puerts 的对接边界。
- 约定核心层与 UE 侧数据交换格式。

## 设计原则

- 尽量少用 Tick 驱动玩法逻辑。
- 采用“命令 -> 事件 -> 状态”的事件驱动链路。
- UE 风格命名：`A* / U* / F* / E* / I*`。
- 单文件控制在 1000 行以内，优先一文件一职责。

## 自动验证策略

1. 单元测试（Vitest）

- 覆盖回合推进、伤害结算、死亡判定等核心流程。

2. 类型检查（TypeScript strict）

- 防止跨层接口漂移。

3. 构建验证（Vite build）

- 确保对外测试版本可打包。

4. 统一入口命令

- 使用 `pnpm verify` 串联类型检查 + 单测 + 构建。
