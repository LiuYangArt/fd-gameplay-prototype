# .codebase_index.md

## 项目概览

- 仓库：`fd-gameplay-prototype`
- 目标：快速验证回合制战斗玩法（Web 3D 原型），核心逻辑可迁移到 UE5（puerts）。
- 包管理：`pnpm workspace`
- 根入口命令：`pnpm dev`（实际启动 `@fd/web-client`）

## 顶层目录索引

- `packages/`：业务代码主目录
- `scripts/`：跨平台开发脚本
- `docs/`：设计、测试、部署与复盘文档
- `.husky/`：Git hooks
- `package.json`：根脚本与工具链配置
- `tsconfig.base.json`：跨包 TypeScript 基础配置（含路径别名）

## 分层与包结构（核心）

### 1) `packages/gameplay-core`（纯逻辑层）

- 职责：玩法规则、回合推进、事件/命令总线、状态管理。
- 入口：`packages/gameplay-core/src/index.ts`
- 关键文件：
  - `src/facade/UBattleSimulation.ts`：玩法门面（高频核心）
  - `src/state/UGameplayStateStore.ts`：状态存取
  - `src/systems/UDamageResolveSystem.ts`：伤害结算
  - `src/systems/URoundOrderSystem.ts`：回合顺序
  - `src/bus/UEventBus.ts`：事件总线
  - `tests/UBattleSimulation.test.ts`：核心回归单测

### 2) `packages/web-client`（表现与输入层）

- 职责：网页运行时、渲染桥接、输入映射、UI 展示。
- 入口：`packages/web-client/src/main.tsx`
- 关键文件：
  - `src/App.tsx`：页面主组件
  - `src/game/UWebGameRuntime.ts`：Web 运行时协调（探索/遭遇/3C/结算状态机）
  - `src/game/USceneBridge.ts`：场景桥接
  - `src/input/UInputController.ts`：输入控制
  - `src/ui/FHudViewModel.ts`：HUD 视图模型
  - `vite.config.ts`：Vite 配置（含 `@fd/gameplay-core` 别名）

### 3) `packages/ue-bridge`（UE 适配层）

- 职责：未来对接 puerts 的桥接层。
- 入口：`packages/ue-bridge/src/index.ts`

## 根脚本速查

- `pnpm dev`：启动 web-client 开发服务器
- `pnpm typecheck`：全仓类型检查
- `pnpm lint` / `pnpm lint:fix`：代码检查/修复

- `pnpm build`：构建全部包
- `pnpm verify`：一键回归（typecheck + lint + test + build）
