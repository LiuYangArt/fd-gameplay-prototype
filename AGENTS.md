# AGENTS.md

## 项目目标

- 本项目用于快速验证回合制战斗 gameplay（3D 网页原型）。
- 验证通过后，核心逻辑要尽量可迁移到 UE5（puerts）。
- 优先迭代效率，其次是迁移一致性。

## 强制规则

1. 所有讨论、计划、任务说明使用简体中文。
2. 核心玩法逻辑必须事件驱动，尽量避免依赖 Tick。
3. 模块与命名遵循 UE 风格：`A* / U* / F* / E* / I*`。
4. 单文件尽量控制在 1000 行以内，保持可读与可维护。
5. 优先小步修改，避免一次性大重构。

## 代码分层约束

- `packages/gameplay-core`：纯逻辑层，不得依赖 DOM、渲染引擎或浏览器 API。
- `packages/web-client`：表现层与输入层，仅负责渲染/UI/输入适配。
- `packages/ue-bridge`：UE 适配层，负责与 puerts 侧数据交互。

## 输入与交互约束

- 必须同时考虑键鼠与手柄交互。
- 输入先抽象为“动作语义”，不要直接散落硬编码按键逻辑。

## 测试与验证

- 提交前至少通过：`pnpm verify`。
- 新增玩法规则时，优先补 `gameplay-core` 单元测试。
- 若修改输入或 UI 关键路径，补最小冒烟验证步骤。

## 工作流程

1. 先确认目标与约束，再动手改代码。
2. 实施时保持“最小可审阅改动”。
3. 完成后给出可复现的本地验证命令。

## 部署约定

- 现阶段仅记录方案，不执行正式部署。
- 部署方案文档：`docs/deployment/hosting-options.md`。

## 开发脚本

- macOS/Linux：`scripts/dev.sh`
- Windows：`scripts/dev.ps1` 或 `scripts/dev.cmd`
